# PalindromeChecker-App
**UseCase1 – Application Entry & Welcome Message**

**Palindrome Checker Management System**

**Overview**

This module represents the entry point of the Palindrome Checker Management System.
It establishes a clear startup flow for the application before any palindrome processing logic is implemented.
At this stage, the application:
1. Starts execution from the main() method 
2. Displays a welcome message 
3. Shows the application version 
4. Does not yet implement palindrome logic 
5. The goal is to define a structured and professional application startup process.

**Application Flow**
1. Program starts. 
2. JVM invokes the main() method. 
3. Welcome message is displayed. 
4. Application version is displayed. 
5. Program exits or proceeds to the next use case.

**Key Concepts in UC1:**
1. Class – Container for the Palindrome Checker application logic. 
2. Main Method – Entry point of the Java application executed by the JVM. 
3. Static Keyword – Allows the main() method to run without creating an object. 
4. JVM (Java Virtual Machine) – Executes the program and invokes the main() method. 
5. Console Output – Displays messages using System.out.println(). 
6. Command-Line Arguments – Passed through String[] args for runtime input. 
7. Application Flow Control – Defines the startup behavior of the program.
